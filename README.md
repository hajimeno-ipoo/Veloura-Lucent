# Veloura Lucent

AI 生成音源などで起きやすい「高域の物足りなさ」と「シュワシュワしたノイズ感」を補正し、そのあと独立したマスタリング機能で仕上げまで行う macOS ネイティブアプリです。補正段では、倍音/シマー解析、高域の foldover 系補完、粒子感を意識した高域ノイズ抑制を組み合わせています。

## 今の構成
- 画面: `SwiftUI`
- 音声読み書き: `AVFoundation`
- 周波数処理: `Accelerate`
- 実行方法: `./script/build_and_run.sh`

Python や Gradio は使っておりません。  
音声処理はすべて Swift 側で完結します。

## 使い方
1. `./script/build_and_run.sh` を実行します
2. アプリで音声ファイルを選びます
3. `補正を実行` を押します
4. 必要なら補正の `ノイズ除去の強さ` を選びます
5. 必要なら `自然 / 配信向け / 前に出す` から仕上がりを選びます
6. 詳細設定で `低域 / 中低域 / プレゼンス帯域 / エアー帯域 / ハーシュネス抑制 / ステレオ幅` を調整できます
7. `マスタリングを実行` を押します
8. 入力ファイルと同じ場所に `*_lifter` と `*_lifter_mastered` が保存されます

## 処理の流れ
1. 音声読み込みと 48kHz への調整
2. 補正段で倍音/シマー解析、ノイズ除去、高域補完、帯域調整
3. `*_lifter` を書き出し
4. 補正済み音源を再解析
5. マスタリング段で de-esser、4バンドEQ、帯域バランス、密度、ステレオ幅、ラウドネスを調整
6. 画面上で入力・補正後・最終版の差を、上段のレーダー/周波数カーブと下段の数値/帯域棒で比較
7. `*_lifter_mastered` を書き出し

## テスト
- `swift build`
- `swift test`

## 注意
- 音質処理はヒューリスティック中心です
- 元音源により効き方は変わります
- 補正段では、倍音量、シマー量、立ち上がり量に応じて高域補完の効き方が変わります
- ノイズ除去は全帯域一律ではなく、高域の粒子感を少し強めに抑える設計です
- 長尺ファイルでは処理時間が伸びます
- マスタリングは補正後音源を前提にしています
